1. Connectivity from Parent client to Reporting Tool (Login):

For connecting the report tool for particular client then,
1. Create the Report Project URL and that URL details add in 'ServiceCredentials.json' file in Client Project.
eg.
{ "default": "http://pcname/RptURL/",
"RedirectHome": "http://pcname/RptURL/Login/",
"LogOut": "http://pcname/RptURL/Home/",
"ParentURL": "http://pcname/ParentClientURL/",
"_Type": "Report_Local"
}
2. First add Report Tag on Header file in Client project.
3. Then After Click to Report tag,in second window open the Reporting Tool with Login.(On Click all logindetails pass from Client to Reporting tool by Url query string data in encrypted form)
4.There are Two Types of Login in Reporting Tool.
a) Normal Login - As normal same as client login to reporting login.
b) Cross Login -
In this we are Cross login to Reporting tool,i.e. In Client Project one company login and for Reporting Tool differnt comapny login. for that in Repoting tool Project have one Cross Login.xml file in App_Data folder.if we want to Cross login for any company then Please Add Cross login Company details to that xml file with Company code.
so when connecting Client to the Reporting tool on basis of that cross login details user can login sucessfully for that company.but if want Cross login functionality then only add cross login details to that file otherwise not need to add any details in crosslogin.xml file.

6. Common Steps for Create Simple Report:

For Create new Report Please follw the common Steps:
1) First Create the cshtml and Js in Views and Sripts Folder with Same Nam And Add cshtml in that Report type Controller and All DO also in same class file.
2) Then Create the Same DO in Model.js which is create in Partiular DO in Report.DO class file.
In Model.js have functions for create DO for Angular Functionlity and binding.
e.g.function AccountReportDO() {
AccountReportDO = {
"ArrivalFrom": "",
"ArrivalTo": "",
"MemberId": 0,
"LoginMemberId": 0,
"LoginUserId": 0,
"BookingdateWise":false,
}
return AccountReportDO;
}
3) Then in Created Report Js first declare the common used DO and other variables defind in NgInit() function.
e.g. $scope.AccountReport = AccountReportDO();(this is Account Report DO)
$scope.ReportFilter = ReportFilterDO();(This is complusory declred in js for Reporting Filtering)
$scope.PostBooking = PostBookingDO(); (This is complusory declred in js for Reporting Filtering)
4) For Post Booking section also have common Partial View and Js (PostBooking.js And PostBookingPartial.cshtml) in Report View Only Call Such Partial Viwes for Rending PostBooking.
And for PostBooking function have to call this function
$rootScope.$on("CallPostBooking", function (event) {
setPostMemberDetails(event);
});
In this function have All Postbooking functionlity related to DB Get Data, as load branch,get Aglist by Branch selected etc. e.g. @{
Html.RenderPartial("PostBookingPartial");
}
for Filter and other Popup also rendering by this way(this is at last ).
e.g. @{
Html.RenderPartial("FilterPopupPartial");
}
5) And Futher Controller call from get data is common as other project and Data is store in one List ($scope.MainResultlist- this is main Data List,which is not changed) and also another List($scope.MainResultlistAfterFilter- this is for Filtering Data List which is changed as per Oprations Perform).
6) Now Report Table is display by Datatable.net,which is a plug-in for the jQuery Javascript library.
7) for Datatable.net lib in cshtml page commonly loaded follwing js:
